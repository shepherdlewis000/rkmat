Redirects

All redirects are handled by .htaccess. Cpanel had a problem doing it some way because https redirect would screw with virtual host I think (like at one time we had dev.rkmasonryandtuckpointing.com set up).
All non http is directed to https and all leading www's are stripped. A copy of .htaccess is saved as htaccess-copy in root of web directory.

Note on 404's

I wrote a custom 404 handler that will do a timed redirect to the homepage while passing the original referrer (having the stale link) and the bad link itself in this format: https://rkmasonryandtuckpointing.com/?source=404&referrer=&badPage=https%3A%2F%2Fwhatever.com%2F328

Cpanel names it 404.shtml. I dunno where it actually keeps the file, but I'll put a copy of it in root of web directory in case it's ever needed.
